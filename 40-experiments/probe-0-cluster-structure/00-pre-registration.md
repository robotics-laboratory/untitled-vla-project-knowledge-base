## Hypothesis

Condition-латенты на выходе верхней модели, снятые для коротких сегментов траектории образуют в своём пространстве кластеры, которые:
1. компактны - латенты одного типа примитива плотно сгруппированы
2. переиспользуются между разными инструкциями - латенты одного типа примитива, полученные при разных языковых инструкциях, лежат вместе, а не разъезжаются по инструкции
3. разделяются между разными типами примитивов - латенты разных типов примитивов занимают разделённые области
## Motivation

В рамках экспериментов требуется определить непосредственно меру простоты действия для того, чтобы иметь возможность доставать а затем анализировать правильные латенты, которые могут быть полезны для нижней системы для ориентации. В случае, если латенты образуют кластеры, то всё хорошо и гипотеза подтверждена. 

Латенты - ключ. Самого value пока что нет. Нам важно посмотреть разделимость именно ключей для эффективного retrieval.
## Method

### Data / inputs

Датасет `lerobot/libero`, все четыре сьюта: Goal, Spatial, Object, Long. В анализ берутся только успешные эпизоды, поскольку извлекаемые из них примитивы должны соответствовать корректному поведению.
### Procedure

Из LIBERO берутся успешные эпизоды по всем четырём сьютам. Для каждого эпизода доступны видеопотоки камер, поток действий и языковая инструкция.
Сегментация траекторий на примитивы: каждая траектория режется на короткие сегменты по сигналам из потоков действий и состояний:
* Событие gripper (открыт или закрыт)
* Локальный минимум скорости - `v_min`
* Смена компоненты скорости

Сегменты короче `L_min` кадров сливаются с соседним, чтобы исключить пересегментацию. Параметры `v_min` и `L_min` фиксируются

После получения списка примитивов присваеваем каждому класс (пока что псевдо-разметка)

Для каждого сегмента берётся кадр начала сегмента вместе с языковой инструкцией эпизода, прогоняется через верхнюю модель, снимается condition-латент. Сохраняется запись: латент, тип примитива, task instruction, id эпизода, сьют. Основной вариант - латент кадра начала сегмента, также есть латент как среднее по кадрам сегмента.

### Metrics
* Silhouette score
* Davies-Bouldin index
* Внутрикластерная компактность
* Переиспользование между инструкциями. Ключевая метрика гипотезы. Для фиксированного типа примитива латенты группируются по task instruction. Считается величина a - средний разброс латентов внутри одной инструкции, и величина `b` - разброс между инструкциями, то есть средняя дистанция между центроидами разных инструкций для того же типа. Метрика `R = b / a`. Значение R, близкое к 1, означает, что инструкция почти не влияет и примитив переиспользуется. Значение R, много большее 1, означает, что латент примитива разъезжается по инструкции и переиспользования нет
* ARI и AMI между кластеризацией латентов и метками типов
## Acceptance criteria
silhouette и ARI выше бейзлайна, компактность мала, `R ≈ 1` - подтверждение
silhouette ≈ 0 или ARI ≈ 0 - опровержение

## Failure modes
Пересегментация или недосегментация. Неудачные `v_min` и `L_min` приводят к тому, что сегменты не соответствуют примитивам, и псевдо-метки шага 3 становятся мусорными
