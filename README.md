# Knowledge Base

Before contributing, read this guide.

1. Create your branch named after yourself: `lastname` or `firstname-lastname`
2. Use your branch to track your experiments
3. Name each experiment `{YY-MM-DD}_EXPERIMENT_NAME` (Obsidian tags are also welcome)
4. Write up and commit as you go
5. When an experiment is complete, open a pull request into main
6. Fact-check, then merge

A report can look like this (not required):

## Introduction

State the hypothesis, the baseline model, and the expected effect.

## Results and Evaluation

Final submission score, validation accuracy, training time, peak VRAM.
